Why test?

  • Prove functionality
  • Much faster than testing manually
  • Makes refactoring easier
  • Acts as documentation
  • Makes your code better

What are acceptance tests?

  • Automated browser based tests
  • Acts as a user of application performing actions
  • e.g. filling out registration form / making an order

Behat

  • Behaviour Driven Development
  • Orientated around communication with stakeholders
  • Build up scenarios and develop against that

What options do we have?

  • Behat with Gherkin (for user stories)
  • Behat with PHPUnit (what we are using)
  • Codeception
  • Laravel Integrated Package

Mink & Goutte

  • Mink is a browser emulator
  • Goutte is a web scraper

{
    "behat/mink": "^1.6",
    "behat/mink-goutte-driver": "^1.1"

Scenario

  • Testing a multi-step registration form
  • Slow and fiddly to do manually
  • Has multiple languages
  • Has differing functionality
  • Specific scenario but hopefully useful demonstration

Data Providers

  • Add to docblock
  • Array of arrays
  • Passed as parameters
  • Useful for quickly passing scenarios to a test

Data Provider Example (1)


/**
* @dataProvider getEuropeanCountryLocales
*/

Data Provider Example (2)


public function getEuropeanCountryLocales()
{
    return [
        ['es'],
        ['fr'],
    ];
